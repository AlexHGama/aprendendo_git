Agora o git é uma ferramenta para gerenciar os arquivos e ao mesmo tempo fazer atualizações dos códigos ,não deixando entrar em conflito se uma pessoa ou mais fez as atualizações no mesmo código , existe uma forma de repositório central que é enviado os códigos a esse repositório tem desvantagem é que se cair o servidar fica sem acesso aos códigos,e tem o modelo distribuido controle de versão se por a caso o servidor caia não para o trabalho de atualizações  por ter acesso as maquinas dos colegas dev ,como uma sincronia entre os desenvolvedores acontece de repositório a repositório e não existe, em principio,um repositório mais importante que o outro,embora o papel de um repositório central possa ser usado para convemcionar o fluxo de trabalho , não impedindo .Todas as operaçoes do Git são locais evitando problemas referente a dependencia de rede ao trafego de arquivos.As informações são asseguradas por um checksum(chave de verificação de integridade), desse jeito o Git tem uma integridade dos seus arquivos.Sendo auxiliado pelos comandos variados que completa essa ferramenta excelente para o desenvolvimento Dev .Exemplos de comandos como, git init - serve para subir os arquivos para o repositorio, git status - mostra os arquivos que tem pra enviar , git add . - adcionar os arquivos para enviar , git status - mostra os arquivos verdes prontos para enviar, para o repositório, git commit -m "texte1" - nomear o commit, git remote add origin "link"-direcionar e origem para onde vai , copia o link no github e coloca no comando dado, git push origin master - subir arquivo para github e outros que vou aprendendo conforme ir subindo os arquivos a ser desenvolvidos.
